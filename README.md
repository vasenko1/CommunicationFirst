# Communication First — Project Working Docs

A lightweight, browser-based, two-person voice/video app (WebRTC).
Free MVP, no TURN, three browsers only (Chrome, Edge, Safari).

## How to use these docs

You are building this **with an AI assistant** (GPT on your Mac) and
**sanity-checking its decisions with Claude**. These files are the shared
context so any assistant is up to speed in one message.

Read / use in this order:

1. **docs/DECISIONS.md** — the fixed decisions. Paste this to GPT at the
   start of every new session so it never drifts off-plan.
2. **docs/ROADMAP.md** — the phased build plan with copy-paste prompts.
   Work one phase at a time. Do not skip ahead.
3. **docs/VERIFICATION.md** — how to prove each phase actually works,
   including the real-network measurement table (the whole point of the MVP).
4. **docs/REVIEW-CHECKLIST.md** — red flags to watch for, and exactly what
   to bring back to Claude to double-check GPT's technical choices.

## Golden rule

Never advance to the next phase until the current one is verified
**on real people in real networks** — not two browser tabs on one machine.
Two-tab tests always pass and tell you nothing.
